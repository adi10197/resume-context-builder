# Common Resume Rules

These rules apply to all resume types unless specifically overridden in individual resume type files.

## Length Constraints
- Maximum length: 2 pages (1 page preferred for <5 years experience)
- Maximum 4-6 bullet points per position
- Less bullet points for experience older than 5 years
- Each bullet point should be 1-2 lines maximum

## Formatting Guidelines
- Use clean, professional markdown formatting
- Use consistent heading levels (# for name, ## for sections, ### for subsections)
- Use bullet points (-) for lists
- Use **bold** for emphasis on key achievements, metrics, and technologies
- Use *italic* sparingly for position titles or dates

## Content Rules

### General
- Use action verbs to start each bullet point (Developed, Implemented, Led, Optimized, etc.)
- Avoid re-using the same action verbs
- Include quantifiable metrics wherever possible (X%, $Y, Z users, etc.)
- Focus on impact and outcomes, not just responsibilities
- Tailor content to be relevant to the target role
- Remove outdated or irrelevant technologies
- Do not bold any part of the bullet points for the roles

### What to Include
- Relevant work experience (last 10-15 years typically)
- Key technical skills and technologies
- Measurable achievements and impact
- Leadership and mentoring experience
- Relevant certifications and education

### What to Exclude
- Personal pronouns (I, me, my)
- Irrelevant hobbies or interests
- Full mailing address (city/state is sufficient)
- References or "References available upon request"
- Objective statements (use summary if needed)
- Reasons for leaving previous positions
- Salary information

## Language Style
- Use present tense for current roles
- Use past tense for previous roles
- Be concise and direct
- Avoid jargon unless industry-standard
- Use active voice over passive voice
- Maintain professional tone throughout

## Sections Order (Default)
1. Header (Name and Contact Info)
2. Professional Summary (optional, use for career changers or senior roles)
3. Technical Skills (for technical roles)
4. Professional Experience
5. Education
6. Certifications (if applicable)
7. Notable Projects (optional)

## Contact Information
- Full name
- Professional email address
- Phone number
- LinkedIn profile URL
- GitHub/Portfolio URL (for technical roles)
- City, State (optional)

## ATS Optimization
- Use standard section headings
- Avoid tables, graphics, or complex formatting
- Include keywords from job description naturally
- Use standard fonts (if converting to PDF/DOCX)
- Avoid headers and footers for critical information
- Use standard file names (FirstName_LastName_Resume.pdf)
