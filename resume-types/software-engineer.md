# Software Engineer Resume Type

This resume type is optimized for software engineering positions with focus on technical skills and development experience.

## Specific Rules

### Target Audience
- Software Engineer positions (Junior to Senior level)
- Backend, Frontend, or Full-stack roles
- Technical individual contributor roles

### Emphasis Areas
1. **Technical Skills** - Prominently featured near the top
2. **Engineering Impact** - Focus on technical achievements and metrics
3. **Technologies** - Highlight relevant tech stack for each role
4. **Code Quality** - Emphasize testing, code review, and best practices

### Section Order
1. Header (Name and Contact Info)
2. Technical Skills (grouped by category)
3. Professional Experience
4. Education
5. Projects (optional - for early career or impressive side projects)
6. Certifications (optional)

## Technical Skills Section Format

Group skills into categories:
```markdown
## Technical Skills

**Languages**: Python, JavaScript, TypeScript, Java, Go
**Frameworks**: React, Node.js, Django, Spring Boot, FastAPI
**Databases**: PostgreSQL, MongoDB, Redis, MySQL
**Cloud/DevOps**: AWS, Docker, Kubernetes, CI/CD, Jenkins
**Tools**: Git, JIRA, VS Code, Postman
```

## Experience Bullet Point Guidelines

### Prioritize
- System design and architecture contributions
- Performance improvements (with metrics)
- Feature development and delivery
- Code quality initiatives (testing, reviews, refactoring)
- Debugging and problem-solving achievements
- Scalability improvements

### De-emphasize
- People management (unless relevant)
- Non-technical meetings or coordination
- Business strategy discussions

### Example Bullet Points
- Developed RESTful API serving 1M+ requests/day using Python and FastAPI, reducing latency by 40%
- Implemented Redis caching layer that decreased database load by 60% and improved response times
- Built CI/CD pipeline with GitHub Actions, reducing deployment time from 2 hours to 15 minutes
- Refactored legacy codebase to improve test coverage from 45% to 90% using Jest and pytest
- Optimized SQL queries and added database indexes, resulting in 3x faster page load times

## Technologies Section per Role

For each position, include a brief tech stack mention:
```markdown
**Technologies**: Python, Django, PostgreSQL, Redis, AWS, Docker
```

## Projects Section (Optional)

Include if:
- Early career (<3 years experience)
- Career gap needs filling
- Side projects demonstrate relevant skills
- Open source contributions are notable

Format:
```markdown
## Projects

**Project Name** | [GitHub Link]
Brief 1-line description, key technologies used
- Achievement or impact metric if available
```

## Job Description Tailoring

When job description is provided:
- Match technology keywords from job posting
- Emphasize experience with required skills
- Highlight similar problem domains
- Use similar terminology from job description
- Feature relevant metrics and scale indicators

## What to Minimize
- Management and leadership (unless specifically relevant)
- Business development or sales activities
- Non-technical soft skills (put focus on technical)
- Extensive education details (keep brief unless recent grad)
