# Solutions Architect Resume Type

This resume type is optimized for Solutions Architect, Sales Engineer, and Technical Consultant positions with focus on customer-facing technical work.

## Specific Rules

### Target Audience
- Solutions Architect positions
- Sales Engineer / Pre-Sales Engineer roles
- Technical Account Manager positions
- Customer Engineer roles
- Technical Consultant positions
- Field Engineer roles

### Emphasis Areas
1. **Customer Success** - Implementations, adoption, satisfaction metrics
2. **Technical Expertise** - Architecture design, integration patterns, best practices
3. **Communication** - Presentations, demos, technical writing, stakeholder management
4. **Sales Impact** - Deal support, revenue influence, pipeline contribution
5. **Problem Solving** - Custom solutions, troubleshooting, technical advisory
6. **Relationship Building** - Executive relationships, trusted advisor status

### Section Order
1. Header (Name and Contact Info)
2. Professional Summary (recommended - highlights customer-facing + technical blend)
3. Professional Experience
4. Technical Skills
5. Certifications (important - vendor certifications highly valued)
6. Education

## Professional Summary Format

Include a 2-3 sentence summary highlighting customer-facing technical expertise:
```markdown
## Professional Summary

Solutions Architect with 8+ years of experience designing and implementing enterprise solutions for Fortune 500 clients. Combines deep technical expertise in cloud architecture with consultative selling skills to drive $20M+ in annual revenue. Known for translating complex technical concepts into business value and building lasting customer relationships.
```

## Technical Skills Section Format

Balance technical depth with customer-facing skills:
```markdown
## Skills

**Architecture**: Cloud architecture, microservices, API design, integration patterns, security architecture
**Cloud Platforms**: AWS (Solutions Architect Professional), GCP, Azure
**Technologies**: Kubernetes, Terraform, Python, REST APIs, GraphQL
**Customer-Facing**: Technical presentations, solution design, RFP response, POC development, executive communication
**Sales**: Discovery, value selling, competitive positioning, deal strategy, pipeline management
**Tools**: Salesforce, Lucidchart, Miro, Confluence, demo environments
```

## Experience Bullet Point Guidelines

### Prioritize
- Revenue and deal influence ($X influenced, Y deals closed)
- Customer success metrics (adoption, satisfaction, retention)
- Technical solution design and implementation
- Presentations and demos delivered
- POC/pilot successes
- Strategic account growth
- Cross-functional collaboration (sales, product, engineering)
- Thought leadership (blog posts, conference talks, best practices)

### De-emphasize
- Pure engineering work without customer context
- Internal-only projects
- Administrative tasks
- Non-customer-facing technical work

### Example Bullet Points (Revenue & Sales)
- Supported $30M in annual sales by providing technical leadership on 50+ enterprise deals with 80% win rate
- Delivered 100+ technical presentations and demos to C-level executives, directly influencing $15M in closed revenue
- Developed competitive differentiation strategy and battle cards, improving win rate against primary competitor from 40% to 65%
- Built and delivered POCs for 25 strategic accounts, achieving 90% conversion to paid customers

### Example Bullet Points (Customer Success)
- Designed and implemented solutions for 40+ enterprise customers, achieving 95% customer satisfaction score
- Led technical onboarding for company's largest customer ($5M ACV), achieving full production deployment in 8 weeks
- Created implementation best practices and reference architectures adopted by 100+ customers, reducing time-to-value by 50%
- Served as trusted technical advisor for 10 strategic accounts, contributing to 120% net revenue retention

### Example Bullet Points (Technical)
- Architected hybrid cloud integration patterns enabling customers to connect legacy systems with modern APIs
- Designed multi-region disaster recovery solution for financial services customer meeting regulatory requirements
- Built reusable demo environment and sample applications, reducing POC setup time from 2 weeks to 2 days

## Certifications Section

Certifications are highly valued in SA roles:
```markdown
## Certifications

- AWS Solutions Architect Professional (2024)
- Google Cloud Professional Cloud Architect (2023)
- Kubernetes Administrator (CKA) (2023)
- [Vendor] Certified Solutions Architect (2024)
```

## Technologies Section per Role

For each position, include customer-facing context:
```markdown
**Focus Areas**: Enterprise integrations, cloud migration, API strategy
**Technologies**: AWS, Kubernetes, Terraform, Python, REST APIs
```

## Job Description Tailoring

When job description is provided:
- Match technical domain (cloud, data, security, etc.)
- Emphasize relevant industry experience (financial services, healthcare, etc.)
- Highlight deal size and customer segment experience (enterprise, mid-market, SMB)
- Feature relevant vendor/platform expertise
- Match pre-sales vs post-sales focus
- Emphasize presentation and communication skills

## What to Minimize
- Pure engineering work without business context
- Internal tooling projects
- Non-customer-facing achievements
- Overly technical details without business impact

## What to Highlight by SA Type

### Pre-Sales SA
- Deal support and win rates
- Competitive positioning
- Demo and presentation skills
- POC delivery

### Post-Sales SA / Customer Engineer
- Implementation success
- Adoption metrics
- Customer satisfaction
- Expansion revenue

### Technical Account Manager
- Relationship building
- Account retention
- Strategic account growth
- Executive relationships

## Metrics to Highlight

- Revenue influenced/closed ($)
- Win rate (%)
- Customer satisfaction (CSAT, NPS)
- Deals supported (#)
- POC conversion rate (%)
- Time-to-value improvements
- Net revenue retention (%)
- Presentations/demos delivered (#)
